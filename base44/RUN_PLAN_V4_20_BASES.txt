GLOBAL RULES FOR EVERY BASE:
- Use config-driven architecture.
- Use storage.js or existing state layer as SSOT for local-first builds unless Supabase/backend is specifically requested.
- Keep demo mode and live mode separated.
- Demo data must be globally disableable.
- Live mode must not depend on demo data.
- No backend-only secrets in frontend code.
- AI agents must be bounded, advisory, and transparent.
- Safety-critical bases require human override and disclaimers.
- Include PWA installability where relevant.
- Include validation gates and rollback notes.
